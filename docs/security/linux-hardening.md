# Linux Security Hardening

## Overview

This document describes the baseline security hardening process applied to the Ubuntu Server.

## Objective

Apply security baseline controls to Ubuntu Server.

## Security Controls

- User account review
- Sudo privilege validation
- Password policy review
- Service review
- System audit preparation

## Security Approach

Audit → Change → Verify

## Verification

- User accounts reviewed
- Sudo privileges verified
- Running services inspected
- Password policy reviewed

## Result

Baseline security controls were successfully implemented to improve the overall security posture of the Linux server.

## Lessons Learned

Applying security baselines early in deployment simplifies future administration and reduces security risks.
